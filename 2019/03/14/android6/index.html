1. MANIFEST

모든 어플리케이션은 최상위 디렉토리에 AndroidManifest.xml이라는 이름의 파일이 있다.

  • Manifest 파일 : 시스템에 알려줘야 하는 어플리케이션에 대한 필수 정보를 담고 있음
    • Java 패키지, activity, service, broadcast recevier, component에 대한 정의
    • 다른 어플리케이션 또는 접근 제한이 있는 API에 접근하기 위한 권한에 대한 정의
    • 어플리케이션이 요구하는 API의 최소레벨에 대한 정의
    • 어플리케이션에서 사용하는 라이브러리 파일에 대한 정의

  • <application\> : 어플리케이션을 정의

    • 어플리케이션의 component에 대한 정의
    • 모든 component에 공통적으로 적용되는 속성을 정의

  • <activity> : activity(어플리케이션의 사용자 상호작용을 담당) 정의

    • 모든 activity는 필드에 정의되어야 한다.
    • 그렇지 않을 경우, 실행할 수 없음

  • <intent-filter>

    • Activity, service, broadcast recevie가 반응할 수 있는 intent 종류를 정의
    • 이 필드에 정의된 intent 타입만을 component에서 받을 수 있다.
    • <action>, <category>, <data>를 사용해서 정의한다.

  • <uses-permission>

    • 어플리케이션이 시스템으로부터 받아야하는 권한에 대한 정의
    • 퍼미션은 어플리케이션이 실행되는 동안 사용자에 의해 허가됨

2. UI

  • 안드로이드 어플리케이션의 모든 UI element들은 View와 ViewGroup을 사용하여 구성
  • View : 사용자와 상호작용하는 화면에 어떤 것을 그리는 객체
  • ViewGroup : 인터페이스의 레이아웃을 정의하기 위해 View 또는 ViewGroup 객체들을 담는 객체

  • XML : 레이아웃을 정의하기 위해 사용
    • HTML과 비슷하게 사람이 읽을 수 있는 layout 구조를 제공
    • ex. <TextView>는 UI에 TextView 위젯을 생성, <LinearLayout>은 LinearLayout view group을 생성
